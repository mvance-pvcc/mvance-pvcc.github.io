Programming Languages
Machine Language To apply the stored-program concept, CPUs are designed to recognize instructions encoded as bit patterns. This collection of instructions along with the encoding system is called the machine language.
Debugging Locating and correcting errors in a program is called debugging the program
Assembler Convert mnemonic expressions into machine language instructions.
Assembly Language A mnemonic system for representing programs is collectively called an assembly language.
Translator Translates programs expressed in high-level primitives into machine-language programs.
Compiler Such a translator was similar to the second-generation assemblers, except that it often had to compile several machine instructions into short sequences to simulate the activity requested by a single high- level primitive. Thus, these translation programs were often called compilers.
Interpreter Rather than producing a machine-language copy of a program that would be executed later, an interpreter actually executed a program from its high-level form.
Cross-Platform-Software Software that is independent of an operating system’s design as well as the machine’s hardware design and is therefore executable throughout a network.
Natural Language Languages (such as English, German, and Latin) that evolved over time without formal grammatical analysis.
Formal Language Languages (such as programming languages) that are precisely defined by grammars.
Machine Independence Not reliant on the characteristics of a particular machine.
High-Level Programming Language Provide more abstractions for the programmer and make it easier for people to read and write a program.
Object-Oriented-Programming (OOP) A software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects. Together, these objects interact to solve the problem at hand.
Scripting Language These languages are typically used to perform administrative tasks rather than to develop complex programs.
Data Types: Integer - Refers to numeric data consisting of whole numbers, probably stored using two’s complement notation.
Float - Refers to numeric data that might contain values other than whole numbers, probably stored in floating-point notation.

Primitive Data Type The data types that are included as primitives in a programming language, such as int for integer and char for character.
Data Structure The conceptual shape or arrangement of data.
Python Collection Data Structures: List - Lists are used to store multiple items in a single variable and are ordered, changeable, and allow duplicate values. Lists are created using square brackets.
Tupple - A tuple is a collection which is ordered and unchangeable. Tuples are written with round brackets.
Set - A set is a collection which is unordered, unchangeable*, and unindexed. Sets are written with curly brackets.
Dictionary - A dictionary is a collection which is ordered*, changeable and do not allow duplicates.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local Variable Aa variable declared within a function is a local variable, meaning that it can be referenced only within that function.
Global Variable Variables whose scopes are not restricted to a particular part of a program.
Source Program The process of converting a program from one language to another is called translation. The program in its original form is the source program.
Object Program The process of converting a program from one language to another is called translation. The translated version is the object program.

All About GitHub

  1. Q. What is a version control system?

    • A. Version control systems are software that help track changes make in code over time.

  2. Q. What is Git?

    • A. Git is a distributed version control system, which means that a local clone of the project is a complete version control repository.

  3. Q. What is GitHub?

    • A. GitHub is a website and cloud-based service that helps developers store and manage their code, as well as track and control changes to their code.

  4. Q. What is the difference between Git and GitHub?

    • A. A Git is a distributed version control system for tracking changes in source code and GitHub is a Git repository hosting service.

  5. Q. Who started GitHub and how was it started?

    • A. Github was launched in 2008 by Tom Preston-Werner, Chrs Wanstrath, P.J. Hyett, and Scott Chacon as a bootstrapped start-up buisiness.

  6. Q. What company owns it now?

    • A. Microsoft

  7. Q. How Much does a GitHub account cost?

    • A. Free for a basic account.

  8. Q. What is the Octocat?

    • A. The mascot of GitHub

Git/GitHub Terms

  1. Repository: A central location in which data is stored and managed.
  2. Commit: A repository snapshot at a specific point in time.
  3. Fork: A fork is a copy of a repository. Forking a repository allows experimentation without affecting the original project.
  4. Push: The git push command is used to upload local repository content to a remote repository.
  5. Pull Requests: Pull requests let you tell others about changes you've pushed to a branch in a repository on GitHub.
  6. Workflows: A Git workflow is a recipe or recommendation for how to use Git to accomplish work in a consistent and productive manner.
  7. Issues: GitHub Issues is a tracking tool that is integrated with your GitHub repository. Use GitHub Issues to focus on important tasks and keep plans up to date simultaneously.
  8. Raw Button: The Raw button, like the name suggests, opens the file in a raw form, meaning that any HTML formatting disappears.
  9. Blame Button: The git blame command is used to examine the contents of a file line by line and see when each line was last modified and who the author of the modifications was.
